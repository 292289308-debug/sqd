## 3. 关键表设计

### 3.1 stocks（股票基础信息）
```sql
CREATE TABLE stocks (
    ts_code      VARCHAR(16) PRIMARY KEY,    -- 000001.SZ
    symbol       VARCHAR(16) NOT NULL,
    name         VARCHAR(64) NOT NULL,
    industry     VARCHAR(64),
    list_date    DATE,
    market       VARCHAR(8) NOT NULL,        -- SZ / SH / HK
    is_active    BOOLEAN DEFAULT TRUE,
    updated_at   TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_stocks_industry ON stocks(industry);
CREATE INDEX idx_stocks_market ON stocks(market);
```

**数据源**: Tushare `stock_basic` 同步（每日）

### 3.2 kline_daily（日 K 线 hypertable）
```sql
CREATE TABLE kline_daily (
    ts_code        VARCHAR(16) NOT NULL,
    trade_date     DATE NOT NULL,
    open           NUMERIC(12,4) NOT NULL,
    high           NUMERIC(12,4) NOT NULL,
    low            NUMERIC(12,4) NOT NULL,
    close          NUMERIC(12,4) NOT NULL,
    pre_close      NUMERIC(12,4),
    change         NUMERIC(12,4),
    pct_chg        NUMERIC(8,4),
    vol            BIGINT,
    amount         NUMERIC(18,4),
    turnover_rate  NUMERIC(8,4),
    pe             NUMERIC(12,4),
    pb             NUMERIC(12,4),
    PRIMARY KEY (ts_code, trade_date)
);
SELECT create_hypertable('kline_daily', 'trade_date',
       chunk_time_interval => INTERVAL '1 year');
```

**重要**: hypertable 不支持外键，所以 `ts_code` 不 FK 到 `stocks` 表，由应用层保证。

**存储估算**: 5000 只股票 × 20 年 × 250 天/年 × 100 字节 ≈ 25 GB（不压缩）
TimescaleDB 压缩后约 5 GB。

### 3.3 strategies（策略定义）
```sql
CREATE TABLE strategies (
    id           BIGSERIAL PRIMARY KEY,
    user_id      BIGINT NOT NULL,
    name         VARCHAR(128) NOT NULL,
    description  TEXT,
    category     VARCHAR(32),               -- trend / mean_reversion / factor
    code         TEXT,                      -- Python 策略源码
    params       JSONB DEFAULT '{}',
    is_public    BOOLEAN DEFAULT FALSE,
    is_template  BOOLEAN DEFAULT FALSE,
    created_at   TIMESTAMPTZ DEFAULT NOW(),
    updated_at   TIMESTAMPTZ DEFAULT NOW()
);
```

### 3.4 backtest_runs（回测任务）
```sql
CREATE TABLE backtest_runs (
    id              BIGSERIAL PRIMARY KEY,
    strategy_id     BIGINT NOT NULL,
    user_id         BIGINT NOT NULL,
    status          VARCHAR(16) DEFAULT 'pending',
    start_date      DATE NOT NULL,
    end_date        DATE NOT NULL,
    universe        JSONB,
    benchmark       VARCHAR(16) DEFAULT '000300.SH',
    initial_cash    NUMERIC(18,2) DEFAULT 1000000,
    commission_rate NUMERIC(8,4) DEFAULT 0.0003,
    slippage        NUMERIC(8,4) DEFAULT 0.001,
    -- 结果指标
    total_return    NUMERIC(10,4),
    annual_return   NUMERIC(10,4),
    sharpe_ratio    NUMERIC(10,4),
    max_drawdown    NUMERIC(10,4),
    win_rate        NUMERIC(8,4),
    trade_count     INT,
    -- 详细结果
    equity_curve    JSONB,
    trades          JSONB,
    metrics         JSONB,
    error_message   TEXT,
    started_at      TIMESTAMPTZ,
    finished_at     TIMESTAMPTZ,
    created_at      TIMESTAMPTZ DEFAULT NOW()
);
```

**注意**: `equity_curve` 和 `trades` 存 JSONB 而不是单独表 —— 简单 + 减少 join。  
V1.0 后单独建 `backtest_equity` 和 `backtest_trades` 表方便分析。

## 4. 索引策略

- **高频查询**: `kline_daily(ts_code, trade_date DESC)` —— 拉某股 K 线
- **回测列表**: `backtest_runs(user_id, created_at DESC)` —— 找历史任务
- **订单查询**: `orders(account_id, created_at DESC)` —— 账户订单流
- **资讯全文**: `news USING GIN(ts_codes)` —— 按股票找新闻

---

## 5. 数据保留策略

| 表 | 保留期 | 归档 |
|---|---|---|
| kline_daily | 全部 | 不归档 |
| kline_minute | 5 年 | 5 年后降采样为 30 分钟 |
| backtest_runs | 1 年 | 老结果移到 OSS |
| news | 3 个月 | 老新闻移到 OSS |
| orders | 10 年 | 监管要求 |

---

## 6. 备份与恢复

- **本地**: `pg_dump` 每日 + WAL 归档
- **云端**: Timescale Cloud 自动备份 + Point-in-time Recovery
- **RTO**: < 30 分钟
- **RPO**: < 5 分钟
