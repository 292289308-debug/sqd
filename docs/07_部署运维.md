## 2. 方式 A：本地直跑（最快，5 分钟）

### 2.1 前置
- Python 3.11+
- Node 18+ (可选，仅 Vite 版前端需要)
- pip 已装

### 2.2 步骤
```powershell
# 1. 安装后端依赖
cd C:\Users\Administrator\.openclaw\workspace\sqd\backend
pip install -r requirements.txt

# 2. 生成 demo 数据 (无 Tushare token 时)
cd ..
python scripts\init_demo_data.py

# 3. 启动后端
cd backend
uvicorn app.main:app --host 127.0.0.1 --port 8000

# 4. (新终端) 启动前端
cd C:\Users\Administrator\.openclaw\workspace\sqd\frontend
python -m http.server 5173

# 5. 浏览器打开
# 前端: http://127.0.0.1:5173
# API 文档: http://127.0.0.1:8000/docs
```

## 3. 方式 B：Docker Compose（生产化）

### 3.1 前置
- Docker Desktop 4.0+
- Docker Compose v2+

### 3.2 步骤
```bash
# 1. 克隆代码
git clone https://github.com/xxx/sqd.git
cd sqd

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 填入 TUSHARE_TOKEN

# 3. 启动
docker compose up -d

# 4. 查看日志
docker compose logs -f backend

# 5. 访问
# 前端: http://localhost:5173
# API: http://localhost:8000/docs
# DB: localhost:5432 (sqd/sqd_dev_pwd)

# 6. 停止
docker compose down
```

### 3.3 数据持久化
- `timescale_data` volume: 数据库
- `redis_data` volume: 缓存
- `./backend/data/`: 本地 CSV 缓存（bind mount）

## 4. 系统服务（生产环境）

### 4.1 systemd（Linux）
```ini
# /etc/systemd/system/sqd-backend.service
[Unit]
Description=SQD Backend
After=network.target timescaledb.service redis.service

[Service]
User=sqd
WorkingDirectory=/opt/sqd/backend
ExecStart=/opt/sqd/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
Restart=on-failure
RestartSec=5s
EnvironmentFile=/opt/sqd/.env

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable sqd-backend
sudo systemctl start sqd-backend
```

### 4.2 Windows Service（nssm）
```powershell
# 安装 nssm
choco install nssm

# 注册服务
nssm install SQDBackend "C:\Python311\python.exe" "C:\sqd\venv\Scripts\uvicorn.exe app.main:app --host 0.0.0.0 --port 8000"
nssm set SQDBackend AppDirectory C:\sqd\backend
nssm set SQDBackend AppEnvironmentExtra "DATABASE_URL=postgresql+asyncpg://..."
nssm start SQDBackend
```

## 5. 反向代理 + HTTPS

### 5.1 Nginx
```nginx
server {
    listen 80;
    server_name sqd.example.com;
    
    # HTTPS 重定向
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name sqd.example.com;
    
    ssl_certificate /etc/letsencrypt/live/sqd.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/sqd.example.com/privkey.pem;
    
    # 前端
    location / {
        root /opt/sqd/frontend/dist;
        try_files $uri $uri/ /index.html;
    }
    
    # API 反代
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    # WebSocket
    location /ws/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

### 5.2 Let's Encrypt 自动续期
```bash
sudo certbot --nginx -d sqd.example.com
```

## 6. 监控

### 6.1 Prometheus + Grafana
启动: `docker compose --profile monitoring up -d`

关键指标：
- `http_requests_total{path, status}` — API QPS / 错误率
- `http_request_duration_seconds` — API 延迟
- `process_cpu_seconds_total` — CPU
- `process_resident_memory_bytes` — 内存
- `timescaledb_chunks` — hypertable chunks 数

### 6.2 日志聚合
- 开发: stdout + 文件（loguru）
- 生产: Loki / ELK / 阿里云 SLS

### 6.3 告警
- API 5xx 错误率 > 1% 持续 5 分钟 → 钉钉
- 数据库连接池满 → 钉钉
- 磁盘空间 > 80% → 邮件
- 回测任务失败 → 飞书

## 7. 备份

### 7.1 数据库每日全量
```bash
# /opt/sqd/scripts/backup.sh
pg_dump -h localhost -U sqd sqd | gzip > /backup/sqd_$(date +%Y%m%d).sql.gz
# 保留 30 天
find /backup -name "sqd_*.sql.gz" -mtime +30 -delete
```

### 7.2 配置文件加密备份
```bash
tar czf - /opt/sqd/.env /opt/sqd/secrets/ | gpg -c > /backup/secrets_$(date +%Y%m%d).tar.gz.gpg
```

### 7.3 上传到 OSS
```bash
aws s3 cp /backup/ s3://sqd-backups/$(date +%Y%m%d)/ --recursive
```

## 8. 故障排查

### 8.1 后端起不来
```bash
# 查看端口占用
netstat -ano | findstr :8000

# 查看进程
ps aux | grep uvicorn

# 测试 Python
python -c "import app.main"
```

### 8.2 前端 404
```bash
# 确认后端在跑
curl http://127.0.0.1:8000/api/v1/health

# 浏览器 Console 查 CORS
# Network 标签看 /api 请求状态
```

### 8.3 回测慢 / 超时
- 减少 universe 股票数
- 减少回测时间范围
- 检查数据是否完整（kline API 返回 count）

### 8.4 数据库连接耗尽
```sql
-- 查看活动连接
SELECT * FROM pg_stat_activity WHERE datname='sqd';

-- 杀掉长查询
SELECT pg_cancel_backend(pid);
```

## 9. 性能调优

### 9.1 PostgreSQL
```sql
-- 共享缓冲 (内存 25%)
ALTER SYSTEM SET shared_buffers = '4GB';

-- 工作内存
ALTER SYSTEM SET work_mem = '64MB';

-- 生效
SELECT pg_reload_conf();
```

### 9.2 TimescaleDB
```sql
-- 启用压缩
ALTER TABLE kline_daily SET (
    timescaledb.compress,
    timescaledb.compress_segmentby = 'ts_code',
    timescaledb.compress_orderby = 'trade_date DESC'
);

-- 压缩 1 年前的数据
SELECT add_compression_policy('kline_daily', INTERVAL '1 year');
```

### 9.3 FastAPI
- 用 `uvicorn --workers 4` 多进程
- async 端点（CPU 密集型用 `run_in_executor`）
- Redis 缓存高频 API（5s TTL）

## 11. 升级流程

```bash

./scripts/backup.sh

# 2. 拉新代码
git pull

# 3. 更新依赖
pip install -r requirements.txt --upgrade

# 4. 数据库迁移
alembic upgrade head

docker compose up -d --build
```
