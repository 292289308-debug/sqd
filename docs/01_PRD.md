## 1. 项目愿景

**成为个人投资者与量化研究员首选的本地化一站式行情+策略+实盘看板。**

## 2. 立项决策（已确认）

| # | 问题 | 决策 | 理由 |
|---|---|---|---|
| Q1 | 目标用户 | **B ToC 散户** | 用户基数大，付费意愿中等 |
| Q2 | 市场范围 | **C A+港美** | 散户全球化需求强 |
| Q3 | 部署方式 | **C 私有化** | 合规风险最低，用户信任度高 |
| Q4 | 核心优先级 | **A+B（看盘+回测）** | MVP 聚焦，后续加 C+D |
| Q5 | 数据源预算 | **B 月付 100~500** | Tushare Pro 高级够用 1 个月验证 |
| Q6 | 技术栈 | **A Python** | 量化生态最成熟 |

## 3. 用户画像

### 3.1 个人投资者（占比 60%）
- **场景**: 早上 9:25 看竞价 → 9:30~11:30 盯盘 → 午盘复盘主力资金 → 下午继续盯 → 收盘复盘写笔记 → 晚上跑回测
- **痛点**: 同花顺"信号提醒"功能弱；JoinQuant 不会写代码；通达信指标不够多
- **付费意愿**: 中（99 元/月可接受）

### 3.2 量化研究员（占比 25%）
- **场景**: Python 写策略 → 10 年 A 股日线回测 → 因子分析 → 绩效归因 → 实盘 vs 模拟盘对比 → 周报
- **痛点**: JoinQuant 集群拥挤；VNPY 配置复杂；本地数据维护困难
- **付费意愿**: 高（团队订阅 999 元/月）

### 3.3 中小私募（占比 10%）
- **场景**: 多账户管理 → 实时风控 → TWAP/VWAP 智能订单 → 业绩归因 → 监管报表
- **痛点**: 私募自研系统成本高（百万级）；购买商业软件贵
- **付费意愿**: 高（私有化 5~50 万/年）

### 3.4 高校/学生（占比 5%）
- **场景**: 学习量化 → 跑模拟盘 → 写论文
- **付费意愿**: 低（教育版免费）

## 6. 验收标准

### 6.1 Phase 1 MVP 验收
- [ ] 用户能本地一键启动（docker compose up 或 python/npm）
- [ ] 至少 5 只真实 A 股的 1 年日线数据
- [ ] 至少 3 个内置策略模板（双均线/海龟/布林带）跑通
- [ ] 回测结果包含：年化、夏普、最大回撤、Alpha、交易笔数
- [ ] K 线图能展示 OHLC + 成交量
- [ ] 自选股能添加/删除/分组
- [ ] 回测报告可导出 CSV

### 6.2 V1.1 验收（实盘）
- [ ] 接入 1 家券商模拟盘（华泰 CTP 或 中泰 XTP）
- [ ] 智能订单：限价/市价/TWAP/VWAP
- [ ] 风控规则：单券限额 / 持仓集中度 / 最大回撤熔断
- [ ] 业绩归因：Brinson / Fama-French

### 6.3 V1.2 验收（资讯）
- [ ] 财联社 / 雪球 新闻实时聚合
- [ ] NLP 情感打分（-1 ~ 1）
- [ ] 公告摘要自动提取（关联股票）
- [ ] 热点概念挖掘（每日 Top 10）

## 7. 风险与约束

### 7.1 技术风险
- **Docker 没装**（本机环境）→ 当前用 Python 直跑 + CDN 前端方案
- **pip 网络** 时好时坏 → requirements.txt 锁定版本，必要时 gh-proxy 拉单文件
- **TA-Lib C 扩展** 安装复杂 → MVP 用 pandas 算 MA，TA-Lib 等 Docker 化解决

### 7.2 合规风险（详见 08_合规说明.md）
- ❌ 禁止承诺收益
- ❌ 禁止代客理财
- ❌ 禁止非法荐股
- ✅ 自用看行情 + 回测无需任何资质
- ⚠️ 提供给第三方需"金融信息服务业务"资质

### 7.3 商业风险
- 同花顺 / 通达信免费压制散户付费意愿
- JoinQuant / BigQuant 已占据量化研究员
- 私募对国产开源信任度低

**应对**: 走"本地化 + 私有化 + 开源"差异化路线，靠社区和口碑扩散。
